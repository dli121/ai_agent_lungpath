# 候选模式表设计说明

日期：2026-06-15

## 1. 这张表到底是什么

候选模式表不是患者数据本身，也不是医学诊断标准。

它是一张提前定义好的“可解释事件模式目录”。系统会把患者输入和本地事件数据拿来和这张表匹配。如果匹配成功，系统就输出：

- 匹配到的候选模式；
- 为什么匹配；
- 支撑证据是什么；
- 还缺什么关键信息；
- 有哪些安全提示；
- 下一步可以如何跟医生或照护人员讨论。

所以它的作用不是回答“这个人得了什么病”，而是回答：

> 这个人的症状、风险背景和医疗事件里，是否出现了值得整理、追问或随访讨论的肺部健康模式？

## 2. 它不是单纯的症状集合

可以把它理解成“症状 + 时间 + 风险背景 + 医疗事件 + 安全解释”的规则目录。

只看症状会太粗糙。例如“咳嗽”本身可能只是短期感染，也可能是长期问题的一部分。候选模式表要把下面这些因素放在一起看：

- 症状：咳嗽、气短、胸痛、喘鸣、咳血、疲劳、体重下降；
- 时间：持续几天、几周，还是反复几个月；
- 风险背景：年龄段、吸烟史、既往慢阻肺、哮喘、支气管炎等；
- 医疗事件：是否反复就诊，是否做过胸片或 CT，是否有急诊记录；
- 用药变化：是否反复使用吸入剂、抗生素或激素；
- 随访障碍：是否错过预约、交通困难、照护困难等。

## 3. 依据从哪里来

候选模式表的依据应该来自四类来源，而不是凭感觉写规则。

### 3.1 公开权威资料

美国疾病控制与预防中心（CDC）列出的肺癌相关症状包括：持续或加重咳嗽、胸痛、气短、喘鸣、咳血、长期疲劳、不明原因体重下降等。

这些症状不能直接用来诊断，但可以帮助我们决定哪些症状字段应该进入候选模式表。

参考链接：  
<https://www.cdc.gov/lung-cancer/symptoms/index.html>

### 3.2 筛查和风险标准

美国预防服务工作组（USPSTF）在肺癌筛查建议中使用了年龄、吸烟包年、是否仍吸烟、戒烟时间等风险字段。

我们的原型不直接做筛查判断，但可以把这些字段作为“背景风险信息”，帮助代理判断是否需要更认真地整理肺部健康时间线。

参考链接：  
<https://www.uspreventiveservicestaskforce.org/uspstf/recommendation/lung-cancer-screening>

### 3.3 疑似癌症识别和转诊指南

英国国家卫生与护理优化研究所（NICE）的疑似癌症识别指南会把年龄、咳嗽、胸痛、气短、体重下降、吸烟史等组合起来考虑。

我们不能把它直接照搬成临床决策规则，但可以用它说明：为什么这些变量在肺部健康随访场景里值得被收集和解释。

参考链接：  
<https://www.nice.org.uk/guidance/ng12>

### 3.4 本项目自己的研究目标

本项目和 IRP 的主线是“诊断前机会图谱”，不是“疾病预测器”。

所以候选模式表要特别关注：

- 症状是否持续；
- 是否反复因为呼吸问题就诊；
- 是否有吸烟史；
- 是否有慢阻肺、哮喘、支气管炎等病史；
- 是否有胸片、CT、用药、复诊缺口；
- 是否有交通、预约、随访困难等社会风险因素。

## 4. 规则应该按什么维度规定

建议按六个维度设计候选模式。

| 维度 | 例子 | 在系统里的作用 |
| --- | --- | --- |
| 症状维度 | 咳嗽、气短、胸痛、咳血、疲劳、体重下降 | 判断当前问题是什么 |
| 时间维度 | 持续几周、反复几个月、逐渐加重 | 区分短暂症状和持续模式 |
| 背景风险 | 年龄段、吸烟史、慢阻肺病史 | 给症状增加上下文 |
| 医疗事件 | 反复就诊、急诊、胸片、CT | 构建诊断前时间线 |
| 用药变化 | 抗生素、吸入剂、激素反复使用 | 反映呼吸问题反复或升级 |
| 随访障碍 | 错过预约、交通困难、照护困难 | 解释为什么后续跟进可能中断 |

## 5. 规则形式

候选模式不应该只有一种形式。第一版可以支持四类规则。

### 5.1 单点危险信号

有些信息不需要组合条件，只要出现就应该先做安全升级提示。

例子：

- 咳血；
- 严重气短；
- 严重胸痛；
- 昏厥；
- 突然意识混乱；
- 症状快速恶化。

这类规则的输出重点不是候选模式解释，而是提醒用户尽快寻求紧急医疗帮助。

### 5.2 两个条件组合

这是最基础的候选模式。

例子：

- 吸烟史 + 持续咳嗽；
- 呼吸系统病史 + 反复气短；
- 疲劳 + 体重下降；
- 呼吸症状 + 近期没有随访。

这类规则适合让代理提出追问和基础随访建议。

### 5.3 三个条件组合

三个条件组合更适合做演示主线，因为解释力更强。

例子：

- 年龄较高 + 吸烟史 + 咳嗽超过几周；
- 慢阻肺病史 + 半年内多次呼吸系统就诊 + 多次用药；
- 呼吸症状 + 体重下降 + 没有近期影像检查记录；
- 气短加重 + 反复就诊 + 吸入剂或激素使用增加。

这类规则可以输出更完整的“为什么值得整理时间线”的解释。

### 5.4 时间序列模式

时间序列模式最贴近 IRP 的诊断前机会图谱。

例子：

> 第一次咳嗽就诊 → 抗生素 → 症状复发 → 再次就诊 → 胸片或 CT

这种模式不是说某一步错了，而是帮助展示症状、就诊、用药、检查之间的先后关系。

## 6. 推荐表结构

第一版候选模式表建议包含这些字段。

| 字段 | 含义 |
| --- | --- |
| 模式编号 | 比如 M001、M002 |
| 模式名称 | 人能读懂的名称，比如“吸烟史 + 持续呼吸道症状” |
| 模式类型 | 危险信号、两个条件组合、三个条件组合、时间序列模式 |
| 必须条件 | 必须满足的字段 |
| 支持条件 | 满足后可以增强解释力的字段 |
| 时间窗口 | 比如过去 3 个月、过去 6 个月、过去 12 个月 |
| 匹配强度 | 低、中、高；只代表解释强度，不代表疾病概率 |
| 证据摘要模板 | 系统如何总结匹配证据 |
| 安全提示 | 明确说明不是诊断、不是临床决策 |
| 下一步建议 | 建议用户如何整理信息、跟医生讨论或寻求帮助 |

## 7. 第一版候选模式目录

第一版建议只做 6 个候选模式，保证范围清楚、可解释、可演示。

| 编号 | 模式名称 | 规则形式 | 匹配条件 | 输出重点 |
| --- | --- | --- | --- | --- |
| M001 | 吸烟史 + 持续呼吸道症状 | 两个或三个条件组合 | 有吸烟史，并且咳嗽、气短或喘鸣持续数周 | 建议整理症状持续时间和吸烟史，用于肺部健康复查讨论 |
| M002 | 呼吸系统病史 + 反复就诊 | 三个条件组合 | 有慢阻肺、哮喘或支气管炎病史，并且近期多次因呼吸问题就诊 | 说明存在反复呼吸系统负担，适合做时间线回顾 |
| M003 | 全身症状 + 呼吸道症状 | 两个或三个条件组合 | 疲劳或体重下降，同时有咳嗽、气短或胸痛 | 建议尽快和临床人员沟通持续性症状，但不判断具体疾病 |
| M004 | 呼吸症状 + 影像检查机会 | 时间序列模式 | 呼吸症状之后出现胸片、CT 或相关检查记录，或长期症状但没有近期影像检查记录 | 解释时间线里何时出现过检查机会，或哪些信息还缺失 |
| M005 | 反复用药或用药升级 | 时间序列模式 | 多次抗生素、吸入剂或激素使用，且与呼吸症状或就诊接近 | 说明可能存在反复或加重的呼吸问题，适合结合就诊记录查看 |
| M006 | 随访障碍或社会风险因素 | 两个或三个条件组合 | 有呼吸症状，同时存在错过预约、交通困难、照护困难等 | 输出照护导航建议，而不是疾病推测 |

## 8. 一个具体例子

用户输入：

> 60 多岁，有长期吸烟史，咳嗽 6 周，最近气短，过去半年因为支气管炎看过两次医生。

系统可能匹配：

- M001：吸烟史 + 持续呼吸道症状；
- M002：呼吸系统病史 + 反复就诊。

安全输出应该类似：

> 系统发现了几个值得随访讨论的肺部健康模式：长期吸烟史与持续咳嗽、近期反复呼吸系统就诊。这些模式不能用于诊断，也不能确认肺癌。它们可以帮助整理就诊前信息。建议和医生讨论症状持续时间、吸烟史、既往就诊记录，以及是否需要进一步评估。

不应该输出：

> 你可能有肺癌。

## 9. 设计边界

候选模式表必须遵守这些边界：

- 不输出医学诊断；
- 不输出疾病概率；
- 不判断漏诊；
- 不判断医疗责任；
- 不替代医生；
- 不把公开指南改写成自动临床决策；
- 只用于教育、工程演示和诊断前事件整理。

## 10. 一句话总结

候选模式表就是一张“规则 + 证据 + 解释 + 安全建议”的目录表。

它用来帮助代理发现值得整理和讨论的肺部健康事件模式，而不是用来判断患者得了什么病。
